진행과정

1. 작업 세분화

  • # 메인페이지
  • # 서브페이지1 ( 사업영역 - 크리에이티브 ) + 헤더 + 푸터
  • # 서브페이지2 ( 플레이디 - ABOUT )

업무 담당은 업무량을 고려해서 구현 할 움직임이 많은 ‘메인페이지’ 하나, 내용이 적은 ‘서브페이지1’에서 ‘헤더+푸터’까지 하나, 컨텐츠가 많은 ‘서브페이지2’까지 하나로 위 내용처럼 세 구역으로 업무를 분담

2. 코딩컨벤션 작성 및 공유

작업을 시작하기 전에 폴더명, 폴더위치, 선택자 작성법 등을 맞춰 경로나 merge 후의 혼란을 최소화 하고자 함.

3. header+footer 공유 후 각 페이지에 적용

각 페이지에서 동일하게 적용되는 header+footer를 먼저 공유해 각 페이지에 적용했지만 페이지마다 header+footer의 작동이 달라지는 문제가 발생.

문제 발생시 서로에게 문제를 공유해서 각자 찾아본 방안을 제시해 문제를 함께 해결하고자 함.

4. 각 브랜치에 push후 main브랜치에 merge

작업 완료된 페이지를 각 브랜치에 push하고 main브랜치에 merge해서 각 페이지를 하나로 합쳐 전체적인 작동을 확인했고 header에 문제가 발생한 것을 확인.

header에 마우스를 올렸을때 부드럽게 내려와야 할 서브메뉴의 백그라운드 영역이 작동하지 않았는데 자바스크립트로 작업했던 부분을 css에서 hover로 수정.

프로젝트 결과

  • header + footer

    스크롤시 헤더와 서브메뉴 높이의 변화
    자바스크립트에서 스크롤 이벤트를 작업해 주었으며 아래로 스크롤시 classList.add로 header의 높이가 줄어들고, 화면 위 끝까지 닿으면 classList.remove로 header의 높이가 처음상태로 늘어나도록 함

    서브메뉴 영역
    '서브메뉴1'의 두번째 목록 선택시 '서브메뉴2'의 목록이 선택되는 문제가 발생해서 ‘서브메뉴1’에 mouseover 되어있는 동안은 다른 서브메뉴가 선택되지 않도록 overflow: hidden이 적용되고 mouseout 될 경우 overflow: visible이 적용되도록 함

    위로가기 버튼이 나타나는 위치
    화면 오른쪽 하단에 고정되어 있는 위로가기 버튼이 안보였다가 아래로 스크롤 할때 나타날 수 있도록 자바스크립트에서 offsetTop을 사용해 해당 위치에서 보여지도록 함

    메뉴 버튼
    메뉴 버튼 클릭시 서브메뉴가 화면 전체를 채우고 꺼지도록 자바스크립트에서 toggle로 '.active' 클래스명을 추가/제거 할 수 있도록 함

  • 서브페이지1 (사업영역 - 크리에이티브)

    이미지 컨텐츠 마우스 이벤트
    이미지 위에 마우스를 올리면 텍스트가 나타나고 영역이 줄어들지만 이미지의 사이즈는 커질 수 있도록 css에서 hover와 opacity를 사용함
